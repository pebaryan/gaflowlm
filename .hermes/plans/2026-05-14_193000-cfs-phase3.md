# Phase 3: CFS (Clifford Flow on the Sphere)

## Goal

Implement the full Clifford Flow architecture: multivector embeddings, CARE positional encoding, and Clifford Frame Attention (CFA) replacing the S-arch backbone.

## Components

### 1. CARE Position Encoding (`models/care.py`)

Rotor-based position encoding instead of sinusoidal RoPE.

```
R_pos(i) = exp(½ θ_x(i) B_x + ½ θ_y(i) B_y)
M̃_i = R_pos(i) · M_i · R̃_pos(i)
```

Where `B_x`, `B_y` are learned bivectors in Cl(k,0,0) and `θ_x(i)`, `θ_y(i)` are position-dependent angles (can be learned or sinusoidal).

**Implementation:**
- `CAREPositionEncoding` module: `k` → learned bivectors `B_x`, `B_y` (shape `[1, 1, 2^k]` each, as multivectors)
- Forward: takes positions `[B, L]`, computes rotor `exp(½ θ·B)`, applies rotor sandwich to multivector sequence `[B, L, 2^k]`
- Compatible with `EmbedToClifford` output shape

### 2. Clifford Frame Attention (CFA) Layer

Replaces standard dot-product attention with geometric bilinear attention in Cl(k,0,0).

**Attention:**
```
Q = W_q · M       (multivector linear proj)
K = W_k · M
V = W_v · M
S_ij = ⟨Q_i · K_j⟩₀ / √d_k    (scalar part of geometric product)
A_ij = softmax(S_ij)
```

**Message:**
```
m_ij = A_ij · GeometricBilinear(Q_i, V_j)
     = A_ij · (Q_i · V_j + Q_i ∧ V_j)
out = Σ_j m_ij     (multivector)
```

**Higher-order messages (optional):**
```
m^{(2)}_i = Σ_j m_ij
m^{(3)}_i ∝ m^{(2)}_i · m'^{(2)}_i
out_i = m^{(2)}_i + γ · m^{(3)}_i
```

### 3. CFS Transformer Block

Full multivector transformer block with rotor residuals.

```
1. CARE position encoding (first block only)
2. CFA attention
3. Rotor residual: M_out = R_γ · M_in · R̃_γ  (learned γ per block)
4. FFN on multivector components
5. Multivector normalization: M / √⟨M·M̃⟩₀
```

### 4. CFS Model (`models/cfs_arch.py`)

Wraps the CFS transformer blocks into a full model compatible with the training loop. Outputs bivector velocity fields for the flow-matching loss.

## Integration

- CFS is a full model swap — it replaces the S-arch backbone entirely
- `standalone_train.py` gets `--algo cfs` to instantiate the CFS model
- The CFS model outputs bivector velocity directly (no need for SLERP/log-map in the flow)

## Order of Implementation

1. CARE position encoding (simple, standalone testable)
2. CFA attention layer (test with toy multivector sequences)
3. CFS transformer block (wire CFA + rotor residuals + FFN)
4. CFS model (full forward pass)
5. CFS algorithm (training wrapper for standalone_train.py)
6. Integration test (forward + backward + mini training loop)

## Testing

- Unit test for CARE: position encoding preserves multivector norm
- Unit test for CFA: attention weights are properly normalized, geometric bilinear produces correct grade structure
- Integration test: CFS model forward pass + loss computation + backprop
